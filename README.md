유니티 2022.3.62f3 버전과 C# Visual Studio를 사용하여 구현했습니다.

전투 루프: 적 생성 및 이동, 영웅 공격, 적 체력 감소, 적 사망처리 (보스 x)

  	보스를 제외한 적 생성 -> 이동과 유닛의 공격 및 적의 체력 감소, 적의 사망처리까지 구현했습니다.
  
웨이브: 승리/실패 판정 (지휘관 체력 기반)

  	3번의 웨이브가 있으며, 3번의 웨이브를 모두 성공적으로 막아내면 승리, 
  	웨이브 중 몬스터를 5번 이상 아군 포탈까지 닿게 하면 게임오버입니다.

  	코인이라는 재화를 만들어 몬스터를 쓰러트릴시 10코인, 유닛 배치시 20코인, 
  	승급에 각 50 / 100 코인 소모되며 승급시 공격력과 공격속도가 증가됩니다.

MapManager를 통해 그리드를 논리좌표로 관리하였으며, 
유닛을 놓을수 있는지 검사하여 배치 혹은 재배치시 로지컬그리드 상태를 갱신합니다.

  기본으로 주어진 오브젝트로 배치해 보면 7x11이 가장 이상적으로 판단하여
  해당 오브젝트를 바탕으로, 2D 정수 배열로 맵 상태를 저장했습니다. 
  
  MapManager 하단에 OnDrawGizmos 메서드가 해당 논리 그리드를 Scene뷰에서 가시적으로 확인할수 있도록
  각 셀의 윤곽선을 그려주는 디버그용 메서드입니다. 주석처리 하여 업로드하였습니다. 

  논리 좌표는 아래의 배열로 관리됩니다.
  private int[,] logicalGFrid = new int[MAP_HEIGHT, MAP_WIDTH]
  
  logicalGrid[y,x] == 0 일때는 빈칸으로,
  logicalGrid[y,x] == 1 일때는 유닛이 있는 칸입니다. 

  논리 그리드를 사용하므로써 유닛간 겹침 방지 및 경로를 관리하기 용이합니다.
  또한 모든 타일 기반 로직을 좌표체계에서 관리 가능합니다. 


Sponer를 통해 웨이브 진행이 됩니다. 웨이브 마다 0.5초 간격으로 프리펩을 10개 생성하며, 
마지막 웨이브가 끝나면 allWavesFinished를 통해 더이상의 생성을 막습니다. 

Enaemy를 통해 맵의 경로를 따라 transform.postion으로 이동하고, 
유닛의 공격을 받아 체력이 0이 되면 파괴됩니다.
아군의 포탈(하단의 오브젝트)에 닿으면 10의 피해를 주고 파괴됩니다.
유닛에게 공격받아 파괴되거나, 아군 포탈에 도달하여 파괴되면 monsterAlive를 감소시켜 승리 조건을 체크합니다.

Unit의 사거리트리거용 콜라이더와 클릭을 받아 메뉴(승급과 재배치버튼)을 여는 
클릭용 콜라이더를 분리하여 구분, 구현하였습니다. 쿨타임마다 트리거(사거리)에 닿은 Enaemy를 공격합니다.
